# Block 1 Review — Power Conversion + Upper Build

**Scope:** block | **Window:** Weeks 1–6 (2026-05-03 → 2026-06-13) | **Reviewed:** 2026-06-14
**Prior review:** none (first review of this arc)
**Data source:** Supabase `exercise_sets`, `daily_metrics` (source of truth) + bundle plan files
**Bottom line:** The program works when run. Strength moved up on every lift that got trained. The block was lost to a **~2.5-week fall-off (W4–W6) that began right after the most complex week (W3 DNT)** — the exact pattern `profile.md` names as the #1 risk.

---

## 1. Block context

- **Emphasis:** power/reactive conversion (squat on maintenance), upper base for bench + pull-up, side-split foundation.
- **Nutrition phase:** ~300 kcal deficit, target ~188 lb by W6.
- **Deviation in-block:** DNT 2-week OLY specialization grafted onto W3–W4 (heavier snatch/clean at ~80%), displacing two weeks of the planned reactive progression.

**Read:** this is not a programming failure. It is a consistency failure, and it has a specific trigger — complexity/volume spiked at W3 (DNT, 92 sets, restructured Wed, new lifts), and adherence cratered the following week. This is the documented Andy pattern: "complex/long programs caused failure," "falling off for 2–3 weeks when things get complicated."

## 4. What worked

- **Strength responded everywhere it was trained:**
  - Bench 185×5 (W1) → 205×4 (W3) ≈ ~232 1RM equiv — up from 230 baseline.
  - Power Clean 155×5 → 175×3 → 185×2 (W6) — clean upward wave.
  - Power Snatch 75 → 85 → 95. Front Squat (DNT) 245×4 → 265×3.
  - Back Squat maintenance held (295×4) with no strength loss — the maintenance dial is correct.
- **The diagnosis-driven design is sound** — power/reactive emphasis, squat on maintenance. Nothing in the data argues against the thesis.
- **W1–W3 proves the athlete CAN run a 5-day week** at high quality. Capacity is there.

## 5. What didn't

- **Complexity spike → fall-off.** DNT (W3) was the high-water mark for volume and novelty, and the next 3 weeks collapsed. The integration was well-designed on paper but it pushed past the complexity ceiling the profile warns about.
- **Pull-up was the orphan.** The most aggressive arc goal (+80×8) got the least training — only two logged sessions all block. Fri sessions (pull-up day) were the first to drop in the fall-off.
- **No jump measurement discipline.** Sprints/jumps/dunk-touch never made it into the data. The KPI that defines the arc is invisible.
- **Cut + bw tracking both lapsed** at the same point as training.

## 6. Carry-forward decisions (keep / change / drop)

- **KEEP:** the power/reactive thesis; squat-on-maintenance; the core 6-lift weekday template; bench progression (it's working).
- **CHANGE:** **hard-cap complexity for the re-entry weeks.** W07 as currently written has 9–10 items on Mon/Fri (Phase 0 flex grafted on) — this is the same complexity profile that triggered the last fall-off. Trim to ≤6 and treat flex as a separate optional block, not session exercises.
- **CHANGE:** give pull-up a protected slot it can't lose — it's behind and it's a headline goal.
- **DROP (or de-emphasize):** mid-block external program grafts (DNT-style) during a window where consistency is the constraint. Add complexity back only once a 3-week adherence streak is re-established.
- **RESET:** re-baseline now — bodyweight, CMJ, approach touch, side-split. The arc has been running 6 weeks with no anchor for its primary goal.

## 7. Recommendations for the Block 2 plan

1. **Re-entry, not continuation.** Treat W7–W8 as a deliberate, simplified re-on-ramp: 6 exercises max, RPE 6–7, every session winnable. Rebuild the streak before chasing loads. (Profile: "RPE 6 for comeback session, then resume.")
2. **Fix the data pipeline before the program.** The fall-off was invisible in real time because logging stopped. Bodyweight (`daily_metrics`) and jump KPIs need a frictionless capture path or the next fall-off is also silent.
3. **Re-baseline G1 + G3 this week** (CMJ, approach touch, side-split cm). Without these, B2's "approach touch +3 in" target can't be evaluated either.
4. **Sanity-check the two stretch goals** flagged in the plan review: pull-up +80×8 (likely reset to +50–60×5–6) and bench 260.
5. **Advance `active/current-week.md` to W07 / Block 02** — it's still pointing at W03, so the daily prescription is stale.

## 8. Hard signals to surface

- 🔴 **Fall-off pattern confirmed and is the binding constraint** — the program is not the problem; adherence is. Every B2 decision should be filtered through "does this make the week easier to complete?"
- 🔴 **Primary goal (dunk) has no measurement trail** — six weeks in, zero jump data.
- 🟡 **Cut not realized** — bw flat-to-up against a deficit; revisit whether the deficit is real or whether maintenance + consistency is the better B2 play.
- 🟡 **W07 already re-introduces the complexity that broke W4** — intervene before the week runs.
