# Milestone 1 — The Plan

## What this milestone delivers

Andy opens a conversation, confirms his goals, and receives a complete training plan he trusts — from the 18-week arc down to tomorrow's workout with PRs to beat. The plan is stored as Obsidian-compatible markdown files in the repo. No webapp, no text push, no logging — just the plan itself, produced through a structured conversation with AI coaching agents.

**Exit condition:** Andy can open any day of the plan, see his workout with exercises tagged to goals, understand why each rep scheme was chosen, and trace any exercise back to the arc-level goal it serves.

---

## User flow

### Step 1: Ingest athlete profile
**Input:** `athletes/andy/profile.md` (already exists)
**What happens:** The system reads the profile — goals, body comp, lifts, injuries, constraints, history, pitfalls. This is the "character sheet."
**Output:** Confirmed athlete context for the coaching agents.

### Step 2: Arc purpose
**Input:** Athlete states their qualitative goal (e.g., "I want to dunk a volleyball by end of summer")
**What happens:** The head coach agent frames this as an arc with a name, purpose statement, and timeline.
**Output:** Arc purpose confirmed.

### Step 3: Goal selection
**Input:** Athlete profile + arc purpose
**What happens:** Each specialist agent proposes goal options based on the athlete's profile. Each goal has:
- Name (memorable, short — "Sky reach," "Iron ratio")
- Type tag (Vertical, Strength, Consistency, Olympic, Mobility, Body comp)
- Metric: current → target (measurable, yes/no verifiable)
- Cost: training session slots required per week
- Reasoning: why this goal maximally serves the arc purpose, specific to this athlete's profile
- Recommended by: which specialist proposed it

The athlete has a **budget** (sessions/week × 2 slots each = 10 slots for a 5-day athlete). They pick 2-3 goals. Can't pick everything — tradeoffs are real. Unchosen goals carry forward to future arcs.

**Output:** 2-3 locked goals with metrics and targets.
**Alignment checkpoint:** Athlete confirms before proceeding.

### Step 4: Block decomposition
**Input:** Locked goals + athlete profile + arc timeline
**What happens:** The head coach decomposes the arc into 3-5 blocks. Each block:
- Serves 1-2 parent goals
- Has a purpose (qualitative) and 2-3 mini-goals (quantitative, yes/no)
- Has a time window (weeks), with deload every 5th week
- Captures an **athlete state snapshot** (working maxes from Supabase, bodyweight, injury status)
- Includes **programming philosophy** — WHY the rep scheme, volume, and intensity are set the way they are for this block, given the goals, cut status, recovery constraints, and injury considerations
- Defines **session types** for the block (oly + strength, plyo + strength, jump days, etc.)
- Has exit tests (pass/fail tests run at end of block)
- Has failure protocols (what to do if exit tests aren't met)

Blocks that serve different goals in the same time window are merged by the head coach — the athlete sees one coherent block, not parallel tracks.

**Output:** Block timeline with mini-goals and programming philosophy.
**Alignment checkpoint:** Athlete confirms block structure before proceeding.

### Step 5: Weekly decomposition
**Input:** Each block's mini-goals + programming philosophy + athlete constraints
**What happens:** Each block decomposes into weekly plans. Each week:
- Has a purpose and 2-3 mini-goals serving the block's goals
- Contains a 5-7 day split with session focus per day
- Each session has:
  - Exercises with sets/reps/load prescriptions
  - Each exercise tagged to which arc-level goal it serves
  - PRs to beat from previous performance (from database or estimated)
  - Coaching notes (technique cues, injury guards, session caps)
  - Rest periods
- Nutrition targets for the day (calories, protein, meal timing notes)
- Flexibility work embedded in sessions

**Output:** Complete weekly plans for every week of the arc.

### Step 6: Daily view
**Input:** Weekly plan for current week
**What happens:** Extract today's session from the weekly plan. This is the "daily card" — the atomic unit of execution.
**Output:** Today's workout + nutrition. This is what the athlete sees.

## File structure (Obsidian markdown)

All plan assets live under `plans/` in the repo. Each arc gets its own directory.

```
plans/
├── athlete-profile.md              # Character sheet (already exists)
├── arcs/
│   └── 2026-summer-dunk/
│       ├── arc.md                   # Arc purpose, goals, timeline, priority stack
│       ├── goals/
│       │   ├── sky-reach.md         # Goal card: metric, target, reasoning, cost
│       │   ├── iron-ratio.md
│       │   └── iron-habit.md
│       ├── blocks/
│       │   ├── block-01-reset-base.md
│       │   ├── block-02-strength-power.md
│       │   ├── block-03-power-conversion.md
│       │   ├── block-04-realization.md
│       │   └── block-05-consolidation.md
│       └── weeks/
│           ├── week-01.md
│           ├── week-02.md
│           ├── ...
│           └── week-18.md
└── templates/
    ├── arc-template.md
    ├── goal-template.md
    ├── block-template.md
    └── weekly-template.md
```

### Obsidian linking

Files cross-reference with `[[wikilinks]]`:
- `arc.md` links to each goal and block
- Each block links to its parent goals and child weeks
- Each week links to its parent block
- Each exercise in a weekly plan references which arc-level goal it serves

This makes the "why is one link away" principle navigable in Obsidian.

## Agent architecture

### Who does what

Each agent is a system prompt + Claude API call. They don't run autonomously — they're invoked in sequence during the plan generation conversation.

| Agent | Role | Input | Output |
|-------|------|-------|--------|
| **Head Coach** | Synthesizes specialist outputs, resolves conflicts, owns arc/block structure | Athlete profile, arc purpose, specialist proposals | Arc structure, block timeline, merged weekly templates |
| **Strength Coach** | Programs squat, pull, press progressions | Athlete profile, block goals, injury status | Exercise selection, rep schemes, load progressions with reasoning |
| **Sport Coach** | Programs jump training, sprint work, dunk-specific drills | Athlete profile, vertical targets, block goals | Jump sessions, approach drills, plyometric programming with reasoning |
| **Olympic Lifting Coach** | Programs clean, jerk, snatch work | Athlete profile, Olympic lift targets, block goals | Olympic lift sessions, technique progressions with reasoning |
| **Mobility Coach** | Programs flexibility work, prehab, injury modifications | Athlete profile, injury status, flexibility targets | Flexibility protocols, exercise swaps, prehab work |
| **Nutritionist** | Programs calorie phases, macro targets, meal architecture | Athlete profile, body comp targets, training load | Calorie/macro targets per block, cut strategy, meal framework |

### Invocation sequence for Milestone 1

This is a single conversation flow. The athlete (Andy) participates at each checkpoint.

```
1. Load athlete profile
2. Head Coach: frame arc purpose, timeline, coaching team
   → Athlete confirms
3. All specialists: propose goal options (parallel)
   → Head Coach: curate into goal shop with costs + budget
   → Athlete selects 2-3 goals within budget
   → CHECKPOINT: goals locked
4. Head Coach: propose block timeline
   Capture athlete state snapshot (working maxes from Supabase)
   Each specialist contributes block-level programming:
   - Strength Coach: squat/pull schemes, progression rules per block
   - Sport Coach: jump programming per block
   - Olympic Coach: Olympic lift programming per block
   - Mobility Coach: flexibility protocols per block
   - Nutritionist: calorie phases per block
   Head Coach merges into coherent blocks with:
   - Programming philosophy (WHY each rep scheme)
   - Session types (oly+strength, plyo+strength, jump days)
   - Exit tests (pass/fail at block end)
   - Deload placement (every 5th week)
   → CHECKPOINT: blocks confirmed
5. For each block, generate weekly plans:
   Specialists provide week-by-week progressions (Dylan Shannon style)
   Head Coach merges into weekly templates
   Each exercise tagged to arc-level goal + PR to beat populated from Supabase
   → CHECKPOINT: athlete reviews sample week, confirms
6. Write all files to plans/arcs/<arc-name>/
```

### Implementation approach

For Milestone 1, agents are NOT autonomous background processes. They're structured prompts in a conversation flow — either:

**Option A: Single conversation with role-switching**
One Claude conversation where the system prompt shifts per step. The "head coach" prompt generates the arc, the "strength coach" prompt generates squat programming, etc. Simple, no infrastructure needed. The conversation IS the onboarding flow.

**Option B: Multi-agent via Claude Agent SDK**
Each specialist is a separate agent invocation. The head coach agent orchestrates by calling specialist agents and merging their outputs. More modular, better separation of concerns, but requires the agent SDK infrastructure.

**Recommendation: Option A first.** Get the plan generated through a single conversation. The quality of the plan matters more than the agent architecture. Option B is a refactor once the plan is proven.
