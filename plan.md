# Train Repo Cleanup Plan

Strip the repo down to essentials. Everything removed is preserved in git history.

## Steps

1. Remove `docs/archive/` — 8 superseded docs (product-vision, product-framework, product-architecture, layer-design-spec, prd-v0, build-plan, implementation-plan, roadmap-v1). All content lives in `docs/system-architecture.md` now.

2. Remove `skills/` — old OpenAI-based skills (train-log-parser, train-output-templates, trainheroic-migration-prep). No longer relevant to multi-agent architecture.

3. Remove `deliverables/` — one-time intake docs (andy-lee-program-overview.docx, andy-lee-programming-sheet.xlsx).

4. Remove `intake/` — old agent_config.json.

5. Remove `data/` — SQLite database (train.db, train.db-wal), TrainHeroic data export (already migrated to Supabase), and data/migrations/.

6. Remove `dist/` — compiled TypeScript output. Regenerated by `npm run build`.

7. Remove `node_modules/` — reinstall with `npm install` after cleanup.

8. Remove `.tokens/` — cached OAuth tokens.

9. Remove stale deployment scripts:
   - `scripts/bootstrap-droplet.sh`
   - `scripts/deploy-droplet.sh`
   - `scripts/generate_trainheroic_migration_csvs.py`
   - `scripts/import_supabase_rest_csv.py`

10. Remove stale root files:
    - `AGENTS.md` — old farm orchestration config
    - `this-week-meal-plan.md` — stale meal plan
    - `nutrition_tracker.xlsx` — stale tracker
    - `.~lock.nutrition_tracker.xlsx#` — lock file cruft
    - `client_secret_google_calendar.json` — OAuth secrets shouldn't be in repo
    - `client_secret_google_docs.json` — same
    - `client_secret_google_sheets.json` — same

11. Remove empty `plans/reviews/` directory.

12. Remove `cleanup.sh` itself — no longer needed after execution.

## What stays

- `docs/system-architecture.md` — canonical architecture
- `docs/product-roadmap.md` — milestones M0–M6
- `docs/schema.md` — database schema reference
- `plans/` — block plans (12), weekly plans (7), active pointers, templates
- `src/` — CLI source code
- `supabase/migrations/` — one applied core schema migration
- `README.md` — updated
- `package.json`, `tsconfig.json`, `.env`, `.gitignore`, `.claude/`
